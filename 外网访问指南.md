# 🚀 外网访问快速启动指南

让朋友通过外网访问你电脑上的网站！

---

## 📋 前提条件

1. ✅ 电脑已安装 Node.js
2. ✅ 已安装 MySQL 并启动
3. ✅ 已配置好后端数据库连接

---

## 🎯 方案：使用 cpolar 内网穿透（推荐）

### 为什么选择 cpolar？
- ✅ 免费版稳定可用
- ✅ 有 Windows 客户端，操作简单
- ✅ 提供固定域名（免费版会变，但可以重新生成）
- ✅ 无需公网服务器

---

## 📝 第一步：安装 cpolar

1. 访问官网注册账号：https://www.cpolar.com/
2. 下载 Windows 客户端
3. 安装并登录 cpolar

---

## 🌐 第二步：配置隧道（需要两条）

### 隧道1：后端 API 隧道

1. 点击左侧"隧道管理"
2. 点击"创建隧道"
3. 填写配置：
   - **隧道名称**：backend-api
   - **隧道协议**：http
   - **本地地址**：127.0.0.1
   - **本地端口**：3000
4. 点击"创建"
5. 复制生成的公网地址（如：`https://abc123.cpolar.cn`）

### 隧道2：前端页面隧道

1. 再次点击"创建隧道"
2. 填写配置：
   - **隧道名称**：frontend-web
   - **隧道协议**：http
   - **本地地址**：127.0.0.1
   - **本地端口**：5173
3. 点击"创建"
4. 复制生成的公网地址

---

## 🏃 第三步：启动你的网站

### 1. 启动后端（第一个终端）

```bash
cd D:\webtest\backend
npm start
```

看到 `Server running on http://0.0.0.0:3000` 就成功了

### 2. 启动前端（第二个终端）

```bash
cd D:\webtest\frontend
npm run dev
```

看到 `Local: http://0.0.0.0:5173/` 就成功了

### 3. 保持这两个终端窗口开启
- ❌ 不要关闭！
- ❌ 不要最小化（可能会休眠）
- ✅ 可以放在后台运行

---

## 📱 第四步：分享给朋友

**告诉朋友访问地址（前端公网地址）：**

```
https://你的前端隧道地址.cpolar.cn
```

例如：`https://random-name.cpolar.cn`

---

## ⚙️ 高级配置：前后端使用同一域名（可选）

如果想让前后端使用同一个域名，需要配置反向代理：

### 方法1：只创建前端隧道，修改后端 API 配置

1. 只创建前端隧道（5173端口）
2. 修改 `frontend/.env`：
   ```
   VITE_API_BASE_URL=https://你的后端隧道地址.cpolar.cn
   ```
3. 重启前端

### 方法2：使用 Nginx 反向代理（进阶）

在 cpolar 中创建一个隧道，然后用 Nginx 将 `/api` 转发到后端 3000，其他转发到前端 5173。

## 🛠️ 常见问题排查

### 问题1：朋友说无法访问
**解决**：
1. 检查你的电脑防火墙是否允许 Node.js 访问网络
2. 检查 cpolar 客户端是否正在运行
3. 检查前后端服务是否还在运行

### 问题2：可以访问但 API 请求失败
**解决**：
1. 确保后端服务正在运行
2. 检查后端隧道是否正常
3. 查看浏览器控制台的错误信息

### 问题3：地址经常变化
**说明**：免费版 cpolar 的域名会变化
**解决**：
- 每次启动 cpolar 后，将新地址告诉朋友
- 或者升级到付费版获得固定域名

## ⚠️ 重要提醒

1. **安全**：
   - 不要在前端暴露敏感配置
   - 定期修改数据库密码
   - 不要告诉陌生人你的隧道地址

2. **性能**：
   - 免费版有流量限制
   - 同时访问人数过多可能会卡顿
   - 建议只给少量朋友使用

3. **稳定性**：
   - 你的电脑需要一直开着
   - 断网后朋友就无法访问
   - 建议告诉朋友你的使用时间
